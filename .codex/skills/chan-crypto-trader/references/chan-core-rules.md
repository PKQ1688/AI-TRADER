# Chan Core Rules for Crypto

## 1. 目标与边界

- 将缠论形态学和动力学用于加密市场结构分析。
- 输出结构化信号和风险状态，不做自动下单执行。
- 不做主观点位预测，只做结构分类与条件触发。
- 所有判定均基于当下已完成的走势，不对未来做确定性承诺。

## 2. 核心定义

### 2.1 K 线包含处理

- 若相邻两根 K 线互含（一根 K 线的高低点完全在另一根的范围内），则先合并再识别分型。
- 合并方向的确定：假设第 n 根与第 n+1 根存在包含关系，而第 n 根与第 n-1 根不存在包含关系：
  - 若第 n 根高点 > 第 n-1 根高点 → 向上包含。
  - 若第 n 根低点 < 第 n-1 根低点 → 向下包含。
- 合并规则：
  - 向上包含：取两根中高点的较高者为高点，低点的较高者为低点。
  - 向下包含：取两根中高点的较低者为高点，低点的较低者为低点。
- 顺序原则：严格按时间顺序逐根处理。先合并第 n 与第 n+1，得到新 K 线后再与第 n+2 比较，依此类推。不可跨步跳并。
- 注意：K 线包含关系遵守结合律但不遵守传递律——第 1、2 根互含且第 2、3 根互含，并不意味着第 1、3 根互含。

### 2.2 分型

- 顶分型：经过包含处理后的相邻三根 K 线中，中间一根的高点是三根中最高的，且中间一根的低点也是三根中最高的。
- 底分型：经过包含处理后的相邻三根 K 线中，中间一根的低点是三根中最低的，且中间一根的高点也是三根中最低的。
- 顶分型的最高点称为"顶"，底分型的最低点称为"底"。
- 无有效分型不得定义顶底——没有顶分型就没有顶，没有底分型就没有底。
- 分型强度参考（非必须，用于辅助判断置信度）：
  - 弱分型：第一根为长阳/长阴，第二三根为小 K 线，多为中继型。
  - 强分型：第二根带长上影/长阴（顶分型）或长下影/长阳（底分型），第三根无法收复第二根区间一半以上，延续成笔的概率极大。
  - 最强分型：第三根跌破/升破第一根的底/顶，且不能收复第一根区间的一半以上。

### 2.3 笔

- 笔：两个相邻的顶分型和底分型之间构成一笔，方向交替。
  - 上升笔：底分型 + 上升 K 线 + 顶分型。
  - 下降笔：顶分型 + 下降 K 线 + 底分型。
- 笔的最低要求（必须全部满足）：
  1. 必须是一顶一底配对，顶对应底或底对应顶。
  2. 顶分型与底分型之间至少有 1 根独立 K 线（不属于任何分型的元素）。
  3. 一笔至少由 5 根经过包含处理的 K 线构成（含顶底分型各自的中间 K 线）。
  4. 同一笔中，顶分型最高 K 线的区间至少有一部分高于底分型最低 K 线的区间。
- 同性质分型的处理规则：
  - 连续出现两个顶分型：前者低于后者，保留后者（取更高的顶）；前者高于后者，保留前者；相等则暂时都保留。
  - 连续出现两个底分型：前者高于后者，保留后者（取更低的底）；前者低于后者，保留前者；相等则暂时都保留。
- 笔的划分步骤：
  1. 完成 K 线包含处理。
  2. 识别所有符合定义的顶分型和底分型。
  3. 按同性质分型规则筛选：连续顶取最高、连续底取最低。
  4. 筛选后余下的分型，相邻顶底配对即构成一笔。
- 经过以上步骤，所有笔可以被唯一划分。

### 2.4 线段

- 线段：至少由三笔构成，且前三笔必须存在价格重叠区间。
- 线段的笔数一定是奇数。
- 线段方向：从向上笔开始的线段方向向上，从向下笔开始的线段方向向下。

#### 2.4.1 特征序列

- 以向上笔开始的线段 S1X1S2X2...SnXn 中，向下笔序列 X1X2...Xn 为该线段的特征序列。
- 以向下笔开始的线段 X1S1X2S2...XnSn 中，向上笔序列 S1S2...Sn 为该线段的特征序列。
- 将特征序列的每个元素视为一根 K 线（取该笔的高低点为区间），按 K 线包含关系进行处理，得到标准特征序列。
- 特征序列两相邻元素间没有重合区间，称为该序列的一个"缺口"。

#### 2.4.2 线段终结的判定

在标准特征序列中寻找分型（向上线段找顶分型，向下线段找底分型），分两种情况：

- **第一种（无缺口）**：构成分型的第 1 个和第 2 个元素之间不存在缺口。则线段在该分型的极值处结束（顶分型取最高点，底分型取最低点）。
- **第二种（有缺口）**：构成分型的第 1 个和第 2 个元素之间存在缺口。需要从该分型极值开始的反方向笔形成新的特征序列，该新特征序列出现对应的分型（即形成了新线段），才确认原线段终结。注意：第二种情况中，后一特征序列不一定要封闭前一特征序列的缺口，且后一序列的分型不再区分第一种/第二种，只要有分型即可。

#### 2.4.3 线段划分程序

1. 确定特征序列。
2. 对特征序列进行包含处理，得到标准特征序列。
3. 在标准特征序列中寻找顶分型（向上线段）或底分型（向下线段）。
4. 判断属于第一种还是第二种情况。
5. 满足条件则线段终结；不满足则原线段延续。

线段被终结，当且仅当形成新线段。即：线段终结的充要条件是新线段形成。

### 2.5 走势中枢

- 走势中枢：某级别走势类型中，被至少三个连续次级别走势类型所重叠的部分。

#### 2.5.1 形式化定义

设构成中枢的 Z 走势段（与中枢形成方向一致的次级别走势类型）的高低点分别为 gn、dn，则：

- **中枢区间**：`ZG = min(g1, g2)`，`ZD = max(d1, d2)`，中枢区间为 `[ZD, ZG]`。
- **四个扩展指标**（n 遍历中枢内所有 Z 走势段）：
  - `GG = max(gn)` — 中枢最高点
  - `DD = min(dn)` — 中枢最低点
  - `G = min(gn)` — 所有高点中的最低
  - `D = max(dn)` — 所有低点中的最高

#### 2.5.2 走势中枢的三种演化

- **延伸**：后续区间 [dn, gn] 与 [ZD, ZG] 有重叠 → 中枢继续，表现为盘整。延伸不会产生更高级别中枢。
- **新生**：后续三段重叠形成新的同级别中枢，且与前中枢不重叠（即前后中枢的任何波动区间都不重叠）→ 趋势形成。
- **扩展**：新中枢形成后，其围绕波动触及前中枢的波动区间 → 升级为更高级别中枢。

#### 2.5.3 走势中枢定理

- **定理一**：在趋势中，连接两个同级别走势中枢的必然是次级别以下级别的走势类型。
- **定理二**：在盘整中，离开或返回走势中枢的走势类型必然是次级别以下的。
- **定理三（终结定理）**：走势中枢终结，当且仅当一个次级别走势离开该中枢后，其后的次级别回抽走势不重新回到该中枢区间内。
- **中心定理二（前后中枢比较）**：
  - 后 GG < 前 DD → 下跌及其延续。
  - 后 DD > 前 GG → 上涨及其延续。
  - 后 ZG < 前 ZD 且 后 GG >= 前 DD → 形成更高级别中枢。
  - 后 ZD > 前 ZG 且 后 DD <= 前 GG → 形成更高级别中枢。

### 2.6 走势类型

- **盘整**：某完成的走势类型只包含一个走势中枢。
- **趋势**：某完成的走势类型至少包含两个以上依次同向的走势中枢。向上为上涨，向下为下跌。
- **走势必完美（技术分析基本原理一）**：任何级别的任何走势类型终要完成。
- **走势分解定理一**：任何级别的任何走势，都可以分解为同级别"盘整"、"下跌"与"上涨"三种走势类型的连接。
- **走势分解定理二**：任何级别的任何走势类型，都至少由三段以上次级别走势类型构成。
- **走势类型延伸**：
  - 对盘整：中枢形成后，后续走势不断回到中枢，不产生新中枢 → 盘整延伸。
  - 对趋势：同向走势不断产生新的同级别中枢 → 趋势延伸。
  - 走势类型延伸随时可能结束，关键在于是否产生新的走势中枢。

### 2.7 背驰与盘整背驰

#### 2.7.1 基本定义

- **背驰**：在趋势中（至少两个同向中枢），最后一段离开中枢的走势力度弱于前一段同向离开的力度。背驰意味着力度衰竭，但不等价于立即反转。
- **盘整背驰**：在盘整中（单中枢），当下笔或线段比前一同向笔或线段力度弱。
- **背驰-买卖点定理**：任何背驰必然制造某级别的买卖点；任何级别的买卖点必然源自某级别走势的背驰。

#### 2.7.2 结构前提

- 趋势背驰的结构前提：a+A+b+B+c 模型，其中 A、B 为同级别走势中枢，a/b/c 为次级别走势。比较 c 段与 a 段的力度，c < a 则构成背驰。
- 盘整背驰不要求多中枢，在单中枢的走势中枢震荡中，比较相邻同向段的力度即可。
- 没有趋势就没有（真正的）背驰。第一个中枢后出现的力度衰减只能算盘整背驰，不是趋势背驰。

#### 2.7.3 背驰的后果

- 背驰后，理论保证其回拉至少重新回到最后一个走势中枢区间内——这是绝对的，非概率性判断。
- 盘整背驰后，仅保证回拉中枢，不保证趋势反转——可能形成第三类买卖点后继续原方向。
- 背驰的回跌/回升力度与级别相关：大级别末段的小级别背驰可引发暴跌/暴涨；大级别初段的小级别背驰影响有限。

#### 2.7.4 MACD 辅助判断

- 适用场景：A/B/C 三段模型，B 为较高级别中枢，使 MACD 的 DIF/DEA（黄白线）回到 0 轴附近。
- 判断方法：比较 C 段完成时对应的 MACD 柱子面积与 A 段对应的面积。C < A → 标准背驰。
- MACD 参数默认 12/26/9，对大多数级别够用。极端快速走势中（如 1 分钟急促上升），可单独观察柱子伸长高度是否触及乖离极限。
- MACD 仅为辅助。精确判断仍需从走势中枢结构出发。单用 MACD 准确率约 90%+，结合中枢结构理论上为 100%。

### 2.8 区间套

- **理论基础**：区间套方法是走势必完美的重要应用。由于走势可以被唯一分解，买卖点可以通过逐级缩小范围来精确定位。
- **基本原理**：高级别确定方向和背驰区域 → 次级别在该区域内进一步定位 → 更小级别精确找到转折点。
- **操作流程**：
  1. 在高级别识别背驰或走势类型完成的区域。
  2. 在次级别观察该区域内的走势结构，寻找次级别的背驰。
  3. 继续向下一级别缩小范围，直到找到可操作的精确转折点。
- **三种应用场景**：
  - 标准嵌套：逐级缩小，在每个级别都找到对应的背驰。
  - 小背驰引发大转折：小级别的背驰恰好在高级别走势类型完成的关键位置，引发超越该小级别的反转。
  - 重复背驰：同一高级别背驰区域内，出现多次小级别背驰，直到最终有效。
- 加密市场中，区间套特别适用于 4h→1h→15m 或 1d→4h→1h 的级别组合。

### 2.9 中阴阶段

- **定义**：一种走势类型完成后（背驰或盘整背驰确认终结），到新的走势类型确立之前的过渡阶段。
- **特征**：
  - 无一例外表现为不同级别的盘整，即围绕前一走势某部分构成的走势中枢震荡。
  - 这个阶段多空齐杀，不断折腾转换，是最容易导致"逃了顶又被套、抄了底又被震出"的阶段。
  - 级别越大的转折，其后的中阴阶段越长、越难熬。
- **分析方法**：
  - 必须借助前一段走势的走势中枢来辅助分析，不能只看新走势本身（因为新走势类型连第一段线段都可能没走出来）。
  - 关于走势中枢震荡的所有分析方法都可用于中阴阶段。
- **操作原则**：
  - 技术好的：利用中枢震荡做短差。
  - 保守的：等待中阴阶段结束（新走势类型的第三类买卖点出现），再确定操作方向。
  - 中阴阶段结束后，不一定是反转——上涨+盘整+上涨完全合理，上涨+盘整+下跌同样可能。

### 2.10 走势中枢震荡

- **定义**：走势中枢的延伸，等价于任意区间 [dn, gn] 与 [ZD, ZG] 有重叠。
- **震荡中轴**：中枢区间 [ZD, ZG] 的中点 Z = (ZD + ZG) / 2。
- **Zn 监控**：每个次级震荡区间的中点依次记为 Zn。
  - Zn > Z → 震荡偏强。
  - Zn < Z → 震荡偏弱。
  - Zn 的变动一般相对平滑，可用于预估下一个震荡的高低点范围。
- **变盘信号**：
  - Zn 最终一定会超越 ZD 或 ZG（否则不可能出现第三类买卖点）。
  - 首次超越时高度警惕——可能构成第三类买卖点，也可能构成中枢扩展。
  - Zn 缓慢提高但无力突破 ZG → 小心上升楔形诱多。反之同理。
- **走势中枢震荡的卖/买点**：出现在向上/向下离开中枢时产生的盘整背驰处。
- **Zn 数量限制**：不超过 9 个（超过则次级别升级），因此这类监控有明确的边界。

### 2.11 同级别分解

- **定义**：把所有走势按一个固定级别的走势类型进行分解。根据走势分解定理，同级别分解具有唯一性。
- **规则**：
  - 在分解级别上，不定义走势中枢延伸，允许盘整+盘整的连接。
  - 分解级别以下的所有级别，允许走势中枢延伸，不允许盘整+盘整连接。
  - 分解级别以上不考虑——所有走势都按该级别分解。
- **Ai vs Ai+2 机械化比较**：
  - 设 A0 为起始走势类型，后续按同级别分解为 A0+A1+A2+...
  - 比较 Ai 与 Ai+2 的力度（盘整背驰方法）：
    - 若盘整背驰：i+2 为偶数时卖出，为奇数时买入。
    - 若不背驰且 i 为偶数：若 Ai+3 不跌破 Ai 高点，继续持有直到出现背驰或跌破。
    - 若不背驰且 i 为奇数：若 Ai+3 不升破 Ai 低点，继续空仓直到出现背驰或升破。
- **自动换挡**：当小级别的同级别分解积累到足以形成更大级别走势时，可以切换到大级别同级别分解，如同开车换挡。
- 这种方法特别适合加密市场的机械化操作——不区分牛熊，只按固定级别节奏操作。

## 3. 执行流程

1. 校验输入数据完整性，不足时直接输出 `insufficient`。
2. 对主级别和次级别分别执行 K 线包含处理。
3. 识别分型，按笔规则（5 根最低要求、同性质分型筛选）生成笔。
4. 基于特征序列分析，按两种终结情况判定线段。
5. 在主级别识别走势中枢，计算 [ZD, ZG] 及 GG/DD，判断走势类型（盘整/趋势）。
6. 判断当前是否处于中阴阶段——若前一走势类型刚完成且新走势类型未确立，借助前中枢分析。
7. 在次级别判断背驰或盘整背驰，并通过区间套缩小转折范围。
8. 生成三类买卖点候选信号。
9. 应用级别优先和风控规则。
10. 输出 JSON 和中文结论。

## 4. 三类买卖点判定

### 4.1 第一类买卖点

- **B1**：下跌趋势（至少两个向下中枢）中，最后一段离开中枢的向下走势出现背驰。背驰后理论保证至少回拉到最后一个中枢区间内。
- **S1**：上涨趋势（至少两个向上中枢）中，最后一段离开中枢的向上走势出现背驰。
- 用途：转折候选。由于背驰后只保证回拉中枢，不保证完全反转，因此 B1/S1 不直接重仓，需次级别确认。

### 4.2 第二类买卖点

- **B2**：B1 确立后，次级别第一次回抽不破 B1 低点并恢复向上方向。
- **S2**：S1 确立后，次级别第一次反抽不破 S1 高点并恢复向下方向。
- 用途：保守确认型的主执行信号。纯形态学即可识别，不依赖背驰判断。

### 4.3 第三类买卖点

- **B3**：次级别走势向上离开中枢后，首次次级别回抽不重新回到中枢区间内（即中枢终结定理的应用）。
- **S3**：次级别走势向下离开中枢后，首次次级别反抽不重新回到中枢区间内。
- 用途：趋势延续跟随信号。B3 不出现则盘整延续或扩展；S3 不出现则同理。
- 注意：第三类买卖点也是中阴阶段结束的标志之一——新走势类型的方向在此确立。

## 5. 级别冲突处理

- 主级别优先。
- 主级别趋势延续时，次级别逆向信号仅用于减仓或等待，不用于重仓反手。
- 主级别临界转折时（如主级别出现 S1 候选），次级别确认可提高执行权重。
- 冲突高时不输出激进反手结论，优先 `wait` 或 `reduce`。
- 背驰的回跌/回升力度与级别密切相关——日线初段的 5 分钟背驰影响有限，日线末段的 1 分钟背驰可能引发暴跌。必须多级别综合考察。

## 6. 保守确认型规则

- 默认优先 B2/B3 或 S2/S3。
- B1/S1 仅作为候选，需次级别确认后才可升级动作。
- 置信度校准标准：
  - `0.8-1.0`：结构完整，次级别已确认，MACD 辅助一致。
  - `0.5-0.8`：结构部分形成，等待次级别确认中。
  - `0.0-0.5`：仅为候选，结构不完整或存在冲突。
- 中阴阶段内的信号，置信度上限不超过 0.6，除非中枢震荡给出明确的第三类买卖点。

## 7. 风控与禁止项

- 每个信号必须包含触发条件（trigger）与失效条件（invalid_if）。
- 禁止"必涨""必跌""稳赚"等确定性或情绪化叙述。
- 数据不足时只输出 `insufficient`，不硬判。
- 仅分析，不下单，不承诺收益。
- 中阴阶段默认 `wait`，除非有明确的中枢震荡短差机会。

## 8. 加密市场适配说明

- 采用 7×24 连续市场假设，不依赖 A 股时段规则。
- 连续市场特性：
  - K 线基本无跳空（除极端事件），因此线段特征序列的缺口极少出现，线段终结多为第一种情况。
  - 无涨跌停限制，中枢扩展比 A 股更频繁，波动范围更大。
  - 走势中枢的延伸周期可能比 A 股更长（7×24 累积更多次级别波动）。
- 允许常见级别组合：
  - 日内：`4h/1h`、`1h/15m`、`15m/5m`。
  - 波段：`1d/4h`、`1w/1d`。
  - 避免 `1m/5m` 级别——加密市场噪声大，极小级别的交易误差与手续费占比过高。
- 加密市场特有风险：
  - 合约爆仓级联可造成超级别的瞬间走势，此时形态学暂时失效，需等走势稳定后重新从最小级别开始分析。
  - 资金费率极端值可作为辅助参考（类似 MACD 的辅助地位），不作为独立判据。
  - 极端波动后（如闪崩/暴拉），优先标注 `risk.conflict_level = "high"` 并建议 `wait`。
- 级别与时间的关系说明：缠论的级别本质上是结构性的，与时间无关。使用 K 线时间周期划分级别是一种简化的实用近似，在绝大多数情况下足够，但需意识到这不是理论上的严格对应。
